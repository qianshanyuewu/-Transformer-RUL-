# 第2章-步骤5：融合健康指标与 FPT 检查记录

## 检查目标

- 确认累积变换公式是否与文献一致。
- 判断当前健康指标构造方式是否合理。
- 检查 `3σ + 连续点` 的 FPT 检测是否给出可解释结果。

## 前检查

当前 `health.py` 中最需要优先核对的是：

1. 累积变换公式
2. 多特征融合健康指标定义
3. FPT 检测规则

## 客观检查结果

### 1. 当前累积变换公式已修正

文献中的累积变换属于“逐点累加 + 逐点缩放”形式。结合公式排版和文本说明，当前实现已修正为：

- 分子：前 `n` 个特征值的累积和
- 分母：前 `n` 个特征值平方和的平方根

即：

- `c_n = (sum_{a=1..n} f(a)) / sqrt(sum_{a=1..n} f(a)^2)`

简单向量验证：

- 输入：`[1, 2, 3, 4]`
- 输出：`[1.000000, 1.341641, 1.603567, 1.825742]`

这与按上述公式直接计算的结果一致。

### 2. 累积变换对真实特征序列有正向作用

以 `35Hz12kN/Bearing1_2` 的平滑后 `atan_std` 序列为例：

- 变换前单调性：`0.325000`
- 变换后单调性：`1.000000`
- 变换前时间相关性：`0.877426`
- 变换后时间相关性：`0.978090`

这说明累积变换在当前第2章中不是装饰步骤，而是能够明显增强退化趋势表达。

### 3. 当前健康指标定义可以保留

当前健康指标构造方式为：

1. 对筛选后特征先做累积变换
2. 以前 `baseline_window` 个采样点作为健康基线
3. 计算每个时刻相对基线的标准化偏离
4. 对多特征偏离取平均，得到融合健康指标

其本质含义是：

- 当前状态偏离健康基线越大，健康指标越高

这一定义虽然不是唯一方案，但逻辑清楚、可解释性强，适合作为第2章当前版本的融合指标。

### 4. `3σ + 连续3点` 的 FPT 规则可解释

当前 FPT 检测规则为：

- 基线窗口：`10`
- 阈值：`mean(HI_baseline) + 3 * std(HI_baseline)`
- 判定条件：基线之后连续 `3` 个点超过阈值

对 `35Hz12kN` 单工况 5 个轴承的检查结果：

- `Bearing1_1`: `FPT = 17`
- `Bearing1_2`: `FPT = 16`
- `Bearing1_3`: `FPT = 15`
- `Bearing1_4`: `FPT = 16`
- `Bearing1_5`: `FPT = 16`

对应阈值大致位于：

- `2.36` 到 `2.43`

这些 FPT 均出现在健康基线之后，且没有贴着第 `10` 个采样点立即触发，说明当前规则没有出现明显的“基线刚结束就误报”的问题。

## 本步骤结论

1. 累积变换公式已经从错误实现修正为文献一致的逐点累积缩放形式。
2. 当前健康指标可以继续采用“多特征偏离基线的平均 z-score”定义。
3. 当前 `3σ + 连续3点` 的 FPT 规则在代表工况上能给出合理结果。
