# 2.1 数据集与研究对象

本文以 XJTU-SY 滚动轴承加速寿命试验数据集作为研究对象。根据数据集目录与配套说明文档，当前可直接使用的原始数据包含 `35Hz12kN`、`37.5Hz11kN` 和 `40Hz10kN` 三种工况，每个工况下各有 `5` 个滚动轴承，共 `15` 个轴承全寿命退化样本。原始数据存储路径为 `XJT U数据集/extracted/XJTU-SY_Bearing_Datasets`。

XJTU-SY 数据集中，每一次采样对应一个 `csv` 文件，文件按采样时间顺序命名为 `1.csv`、`2.csv` 到 `N.csv`。结合原始文件抽查和数据集说明文档可以确认：每个 `csv` 文件的第一列为水平方向振动信号，第二列为竖直方向振动信号。本文在最终冻结协议中明确以**水平振动信号**作为第2章和第3章共享的主分析对象，不再引入当前数据集中无法直接验证的温度信息，也不再把双通道组合作为正文主线。

进一步统计发现，不同轴承的寿命长度差异显著。以 `csv` 文件数量表征寿命采样长度时，`35Hz12kN` 工况下 5 个轴承的寿命长度介于 `52` 到 `161` 之间，`37.5Hz11kN` 工况下介于 `42` 到 `533` 之间，`40Hz10kN` 工况下则介于 `114` 到 `2538` 之间。三种工况合计共有 `9216` 个采样文件，反映出不同载荷与转速条件下轴承退化进程存在明显差异。

对原始采样文件的全量检查表明，当前 `9216` 个 `csv` 文件全部具有一致的数据长度，每个文件均包含 `32768` 个采样点的双通道记录。这意味着本文后续的信号预处理和特征提取可以在统一采样长度下进行，而轴承之间的主要差异将体现在寿命时序长度不同，而不是单次采样维度不一致。

基于上述事实，本文第2章后续分析将以“原始水平振动信号序列”为统一输入对象：在每个轴承内部，将 `1.csv` 到 `N.csv` 视为随寿命推进的退化观测序列；在不同轴承之间，则保留其寿命长度差异，不对原始寿命时序做先验裁剪。这一处理方式为后续去噪、特征提取、退化表征和健康状态识别奠定了统一的数据基础。
