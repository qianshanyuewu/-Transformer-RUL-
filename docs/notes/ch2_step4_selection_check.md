# 第2章-步骤4：特征平滑与筛选检查记录

## 检查目标

- 判断 SG 平滑是否有必要保留。
- 说明“单调性 + 时间相关性评分”的作用边界。
- 为 Pearson 去冗余选择一个比旧版更合理的阈值。

## 前检查

当前第2章在候选特征池之后，还需要经过三步：

1. SG 平滑
2. 趋势性辅助评分
3. Pearson 去冗余

需要明确的是：

- 评分不等于最终特征选择
- Pearson 阈值不能再回到旧版的 `0.95`

## 客观检查结果

### 1. SG 平滑对趋势指标有正向作用

以 `35Hz12kN/Bearing1_2` 的水平通道代表序列为例，当前 `SG(window=11, polyorder=3)` 平滑后，10 个候选特征的单调性均有不同程度提升，时间相关性多数也有所提升。例如：

- `atan_std`：单调性 `0.112500 -> 0.325000`，相关性 `0.875932 -> 0.877426`
- `rms`：单调性 `0.087500 -> 0.362500`，相关性 `0.851126 -> 0.855342`
- `peak_to_peak`：单调性 `0.087500 -> 0.125000`，相关性 `0.876124 -> 0.918048`
- `freq_entropy`：单调性 `0.012500 -> 0.137500`，相关性 `0.590932 -> 0.605053`

因此，当前可以保留：

- 先平滑
- 再做趋势性评分

### 2. “单调性 + 时间相关性”评分应保留，但只做辅助排序

当前评分定义为：

- 单调性权重：`0.5`
- 时间相关性权重：`0.5`

这一步的作用是：

- 优先把更像“退化轨迹”的特征排在前面

这一步不应该被写成：

- 它直接决定最终特征真值

更准确的角色是：

- 趋势性预排序
- 为 Pearson 去冗余提供优先级

### 3. Pearson 阈值需要明显低于 `0.90`

在新的 20 维双通道候选池上，对 `35Hz12kN` 单工况进行阈值摸底，结果如下：

- `0.80`：保留 `10` 个特征
- `0.85`：保留 `12` 个特征
- `0.90`：保留 `13` 个特征

在 `0.90` 下，被剔除的仍主要是非常明显的同族或跨通道重复特征，例如：

- `h_atan_std` 被 `v_atan_std` 替代，平均 Pearson `0.981431`
- `h_rms` 被 `v_atan_std` 替代，平均 Pearson `0.912003`
- `v_rms` 被 `v_atan_std` 替代，平均 Pearson `0.917492`
- `h_peak_to_peak` 被 `v_atan_std` 替代，平均 Pearson `0.913258`

这说明：

- `0.90` 对当前较小候选池来说仍偏松
- `0.80` 又略显激进

因此，当前先冻结为：

- `Pearson = 0.85`

### 4. 当前筛选逻辑仍然合理，但边界已经更清楚

当前筛选逻辑应理解为：

1. 候选特征先经 SG 平滑
2. 用单调性 + 时间相关性做辅助排序
3. 按排序顺序做 Pearson 去冗余

其中：

- SG 平滑负责减弱局部抖动
- 趋势评分负责给出优先级
- Pearson 负责删除重复表征

三者功能不同，不能混写。

## 本步骤处理

当前配置已明确为：

- `FEATURE_SCORE_WEIGHTS = {"monotonicity": 0.5, "correlation": 0.5}`
- `MAX_FEATURE_CORRELATION = 0.85`

## 本步骤结论

1. SG 平滑保留。
2. 趋势评分保留，但仅作为辅助排序。
3. Pearson 默认阈值从旧口径进一步降到 `0.85`。
