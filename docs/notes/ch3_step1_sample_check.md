# 第3章步骤1：样本构造检查

## 1. 检查目标

验证第3章样本构造代码是否满足当前冻结协议：

1. 以轴承为单位划分 train / val / test
2. 三工况统一使用固定的 `12` 个水平振动累积时域特征
3. 从每个轴承的全寿命序列开始切滑动窗口，而不是从 FPT 之后开始
4. 标签定义为窗口末端对应的寿命比值 `P`
5. 输出对象形状正确，能够直接供后续模型训练使用

## 4. 客观结论

当前样本构造满足协议要求：

1. 训练、验证、测试已经按轴承划分，没有窗口级随机打散。
2. 三个工况统一使用同一组固定 `12` 个水平振动累积特征，不再按工况动态改变输入维度。
3. 所有 split 都成功生成了 `window_size × feature_dim` 形式的样本张量，且 `window_size=10`、`feature_dim=12` 保持一致。
4. 标签已经按照寿命比值 `P` 递减到 `0`，说明窗口末端监督定义是自洽的。

## 5. 当前风险

### 风险1：不同工况样本量差异很大

- `35Hz12kN` 训练样本数为 `370`
- `37.5Hz11kN` 训练样本数为 `1113`
- `40Hz10kN` 训练样本数为 `5333`

这意味着不同工况下训练稳定性和过拟合风险会明显不同，后续结果分析不能简单横向比较绝对误差大小。

### 风险2：短寿命轴承导致验证/测试集仍然较小

- `Bearing2_4` 仅产生 `33` 个验证窗口
- `Bearing1_5` 仅产生 `43` 个测试窗口
- `Bearing3_5` 仅产生 `105` 个测试窗口

这会使单次训练结果的随机波动较大，因此后续至少应使用多个随机种子重复实验。

### 风险3：训练窗口被长寿命轴承主导

例如 `40Hz10kN` 下：

- `Bearing3_1` 贡献 `2514` 个训练窗口
- `Bearing3_2` 贡献 `2472` 个训练窗口
- `Bearing3_3` 仅贡献 `347` 个训练窗口

如果后续直接按窗口均匀采样，模型容易被长寿命轴承主导。后续训练阶段需要考虑：

1. 是否按轴承做均衡采样
2. 是否在损失函数中引入样本权重
3. 是否保留当前原始分布作为最基线方案
