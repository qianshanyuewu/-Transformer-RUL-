# 第2章-步骤1：原始数据集分析检查记录

## 检查目标

- 明确当前第2章所依据的原始数据事实。
- 在进入去噪之前，确认数据集输入对象没有歧义。

## 检查范围

- 数据集目录结构
- 工况与轴承数量
- 单个 `csv` 文件的列含义
- `csv` 文件命名与时序关系
- 各轴承寿命长度分布
- 采样文件长度一致性

## 检查结果

### 1. 数据集目录结构清晰

原始数据位于：

- `XJT U数据集/extracted/XJTU-SY_Bearing_Datasets`

当前目录结构为：

- `35Hz12kN`
- `37.5Hz11kN`
- `40Hz10kN`

每个工况下各包含 5 个轴承文件夹，因此当前原始数据一共包含：

- `3` 个工况
- `15` 个轴承

### 2. 当前原始输入对象是双通道振动 csv

抽查文件：

- `XJT U数据集/extracted/XJTU-SY_Bearing_Datasets/35Hz12kN/Bearing1_1/1.csv`

表头为：

- `Horizontal_vibration_signals`
- `Vertical_vibration_signals`

因此当前可直接确认的输入对象是：

- 第一列：水平方向振动信号
- 第二列：竖直方向振动信号

当前不能据此写入温度数据。

### 3. csv 文件顺序就是采样时间顺序

根据文献：

- `文献/XJTU-SY Rolling Element Bearing Accelerated Life Test Datasets_ A Tutorial.pdf`

可确认：

- 每次采样的数据存放在一个 `csv` 文件中
- 第一列为水平振动，第二列为竖直振动
- 文件按采样时间顺序命名为 `1.csv` 到 `N.csv`

因此，当前第2章与第3章都可以把：

- `1.csv -> 2.csv -> ... -> N.csv`

视为轴承全寿命退化时序。

### 4. 各轴承寿命长度差异很大

逐轴承 `csv` 数量统计如下：

`35Hz12kN`

- `Bearing1_1`: `123`
- `Bearing1_2`: `161`
- `Bearing1_3`: `158`
- `Bearing1_4`: `122`
- `Bearing1_5`: `52`

`37.5Hz11kN`

- `Bearing2_1`: `491`
- `Bearing2_2`: `161`
- `Bearing2_3`: `533`
- `Bearing2_4`: `42`
- `Bearing2_5`: `339`

`40Hz10kN`

- `Bearing3_1`: `2538`
- `Bearing3_2`: `2496`
- `Bearing3_3`: `371`
- `Bearing3_4`: `1515`
- `Bearing3_5`: `114`

按工况汇总：

- `35Hz12kN`: 最短 `52`，最长 `161`，均值 `123.2`，总采样点数 `616`
- `37.5Hz11kN`: 最短 `42`，最长 `533`，均值 `313.2`，总采样点数 `1566`
- `40Hz10kN`: 最短 `114`，最长 `2538`，均值 `1406.8`，总采样点数 `7034`

总计：

- `9216` 个时序采样文件

### 5. 每个采样文件长度一致

抽查多个工况和轴承后，单个 `csv` 文件均为：

- `32768` 行采样数据

进一步全量检查结果为：

- `9216` 个 `csv` 文件全部为 `32768` 行

因此，当前原始数据的单次采样长度是一致的。

## 本步骤结论

当前第2章在进入去噪之前，可以冻结以下输入事实：

1. 只使用 XJTU-SY 原始双通道振动数据。
2. 当前不写温度。
3. 每个 `csv` 对应一个采样时刻。
4. 每个 `csv` 都包含 `32768` 个双通道采样点。
5. `1.csv` 到 `N.csv` 构成全寿命退化时序。
6. 不同轴承寿命长度差异显著，后续处理必须允许变长序列存在。
