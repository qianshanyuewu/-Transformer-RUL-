# 第2章-步骤3：候选特征提取检查记录

## 检查目标

- 确认候选特征提取的输入输出对象是否正确。
- 判断是否需要死守旧版 `13` 个时域特征口径。
- 在进入平滑与筛选之前，先冻结一个更干净的候选特征池。

## 前检查

原始较大候选集虽然可以运行，但已经暴露出明显问题：

- `std / rms / mean_abs / energy` 同属幅值分散类，且高度相关
- `atan_std / asinh_std` 同属变换标准差类
- `peak_to_peak / upper_bound` 同属极值范围类
- `impulse_factor / crest_factor / margin_factor` 同属冲击性指标类

因此，当前不再把“时域特征必须等于 13 个”视为硬约束，而是改成：

- 先按特征家族选代表项
- 再进入后续平滑、评分和 Pearson 去冗余

## 客观检查结果

### 1. 当前特征提取输入输出对象正确

当前输入对象为：

- 单条去噪后的一维振动序列

当前输出对象为：

- 单通道 `10` 维特征向量

抽查结果：

- 时域特征数：`6`
- 频域特征数：`4`
- 单通道总特征数：`10`

对单个双通道采样文件，当前输出为：

- 双通道总特征数：`20`

对单个轴承，当前输出为：

- `20` 条特征序列
- 每条序列长度均等于该轴承的采样文件数量

### 2. 当前保留的候选特征池

当前每个通道保留的时域候选特征为：

- `atan_std`
- `rms`
- `peak_to_peak`
- `impulse_factor`
- `kurtosis`
- `skewness`

当前每个通道保留的频域候选特征为：

- `freq_centroid`
- `freq_peak`
- `freq_band_energy_ratio`
- `freq_entropy`

因此，双通道总候选维度为：

- `20`

### 3. 当前候选池是“按家族留代表”，不是“按数量凑满”

当前删去的特征主要有两类原因：

1. 物理含义与已保留特征高度重叠  
   例如：`std`、`mean_abs` 与 `rms`；`asinh_std` 与 `atan_std`；`upper_bound` 与 `peak_to_peak`

2. 在当前固定采样长度下存在明显函数依赖或近似依赖  
   例如：每个 `csv` 均为 `32768` 点，因此 `energy` 与 `rms` 具有强函数关系，不适合作为并列候选特征长期保留

### 4. 当前实现数值稳定

对 15 个轴承首个采样文件、共 30 条通道序列的检查结果：

- 检查序列数：`30`
- `NaN/Inf` 数量：`0`

说明当前候选特征提取实现不存在明显数值异常。

### 5. 冗余现象客观存在，因此后续去冗余仍有必要

即使在较大的旧候选集中，单轴承内部也能观察到天然高相关特征对，例如：

- `std` 与 `rms`：`0.999999`
- `rms` 与 `mean_abs`：`0.999519`
- `impulse_factor` 与 `margin_factor`：`0.999427`
- `atan_std` 与 `asinh_std`：`0.994703`
- `peak_to_peak` 与 `upper_bound`：`0.970889`

这说明：

- 不是先有 Pearson 去冗余，才“看起来冗余”
- 而是原始候选特征里本来就有不少同族重复

## 本步骤结论

当前第2章不再采用“固定 13 个时域特征”的旧口径，而是冻结为：

1. 每通道 `6` 个时域代表特征
2. 每通道 `4` 个频域代表特征
3. 双通道总候选维度 `20`
