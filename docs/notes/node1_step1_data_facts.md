# 节点1-步骤1：原始数据事实核对

## 检查目标

在冻结第2章流程之前，先确认我们真正拥有的原始数据事实，避免后续方法设计建立在错误假设上。

## 检查内容

### 1. 原始数据列结构

从 XJTU-SY 原始 csv 文件直接检查可知，表头为：

`Horizontal_vibration_signals,Vertical_vibration_signals`

这说明当前原始数据包含两列振动信号：
- 第一列：水平振动
- 第二列：竖直振动

### 2. 采样文件顺序含义

在 `XJTU-SY Rolling Element Bearing Accelerated Life Test Datasets: A Tutorial` 中明确说明：

- 每一次采样对应一个 csv 文件
- 第一列为水平方向振动信号
- 第二列为竖直方向振动信号
- 各个 csv 文件按采样时间先后顺序命名，即 `1.csv`、`2.csv`、…、`N.csv`

因此，在当前论文重构中，每个 csv 文件都应视为一个离散采样时刻。

### 3. 当前桌面重建工程是否与数据事实一致

检查 `data_processing/io.py` 可知：

- `read_csv_file()` 读取第 1 列和第 2 列
- `parse_bearing_folder()` 使用数值排序，保证 `1.csv` 到 `N.csv` 顺序正确
- `load_bearing_signals()` 按排序后的 csv 文件依次加载数据

这与原始数据事实一致。

## 本步骤结论

当前可冻结的原始数据事实如下：

1. 我们现在只有 XJTU-SY 原始双通道振动数据。
2. 当前没有可直接验证的温度列，因此论文现阶段不能写成“振动 + 温度”。
3. 每个 csv 文件对应一个采样时刻，文件编号天然给出了寿命时序顺序。
4. 第2章和第3章都必须建立在“双通道振动时序数据”这个基础上。

## 对后续流程的约束

- 第2章默认采用双通道振动特征提取。
- 第3章样本构造必须基于按时间顺序排列的采样文件序列。
- 后续若要引入温度，必须先提供新的可验证原始数据来源，而不能靠旧论文或旧代码假设。
